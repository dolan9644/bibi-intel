## 重大故障分析（2026-03-24）

### 故障现象
bibi-writer/topic/design 的 SOUL.md 等 bootstrap 文件全部丢失，导致晨报输出"和昨天完全不一样"。

### 根因
1. **git 分支混乱**：存在 11 个本地分支，晨报相关文件只存在于 `brief-2026-03-24` 分支，main 分支从未包含这些文件
2. **cron job 触发切换**：cron 运行时发现 `automation/` 脚本在 main 上不存在，自动切换到 `brief-2026-03-24` 分支读取脚本，导致整个工作目录切换分支
3. **sub-agent workspace 是 untracked 文件**：git checkout 切换分支时，磁盘上的 bibi-writer/topic/design 目录中的文件如果未 commit，就会丢失或被覆盖

### 恢复过程
1. `git checkout brief-2026-03-24 -- bibi-writer/ bibi-topic/ bibi-design/` 恢复 22 个 sub-agent 文件
2. `git checkout brief-2026-03-24 -- automation/` 恢复 6 个 cron 依赖脚本
3. 两次 commit + push 到 main：
   - commit 1: sub-agent bootstrap 文件（22个）
   - commit 2: automation 脚本（6个）

### 防止再犯
1. **git alias `save`**：`git save` 自动 add → commit（日期作message）→ push bibi-writer/ bibi-topic/ bibi-design/ automation/
2. **规范**：这三个目录 + automation/ 任何一个改了，立刻 `git save`
3. **删除旧分支**：只保留 main，删除所有历史分支避免混淆

---

## 已解决：git alias save
配置：`git config alias.save '!git add bibi-writer/ bibi-topic/ bibi-design/ automation/ && git commit -m "$(date +%Y-%m-%d)" && git push'`
使用：`git save` = 一条命令保存所有关键文件到 main
