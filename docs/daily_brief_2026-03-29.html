■ GitHub 训练门
https://api.github.com/repos/OWNER/REPO... · https://news.ycombinator.com/item?id=475... · https://news.ycombinator.com/item?id=475... · https://www.vice.com/en/article/github-a...

2026年3月29日，GitHub悄无声息地在条款中添加了一个致命条款——4月24日后，所有未主动退出（opt-out）的私有仓库代码将自动成为GitHub AI模型的训练语料。消息在HN上炸出298 points、130评论，圈内一片哗然。这是继2023年GitHub Copilot训练数据争议之后，平台与开发者之间最激烈的一次对撞。

这次不是演习。GitHub的新条款本质上是一次大规模的代码资源收割行动。用户即便在4月24日之前没有采取任何行动，其私有代码也将被"默许"用于AI训练——opt-out的设计本身就充满了平台傲慢。开发者社区的反应分为两派：一派认为这是对开源精神的背叛，私有代码不同于公开代码，其商业价值和知识产权属性不可忽视；另一派则认为GitHub为开源社区提供了免费服务，适当的数据回馈是合理的交换。 问题的核心在于"权力不对称"。GitHub（微软系）掌握着数以亿计的私有仓库，这些代码是训练世界顶级代码生成模型的宝贵语料库。在没有任何外部监管的情况下，平台单方面决定将其用于商业AI训练，而用户不仅要承担代码被学习后可能产生的IP风险，还要承担"沉默即同意"这种流氓逻辑带来的心理负担。 更值得警惕的是，这只是大平台数据收割浪潮中的一朵浪花。Anthropic、OpenAI、Google的代码模型背后，都有大量未明确授权的代码在流通。GitHub的开创性动作可能会引发其他平台的效仿，形成整个行业对开发者私有代码的系统性收割。

第一步：登录GitHub，进入Settings → Repository → Repository features → Actions
第二步：进入你的仓库Settings → Pages → 确认没有暴露敏感信息
第三步（最关键）：在仓库根目录创建.github/DISABLE_TRAINING文件，或者通过GitHub API批量设置：
curl -X PATCH -H "Authorization: token YOUR_GITHUB_TOKEN" \
  -H "Accept: application/vnd.github+json" \
  https://api.github.com/repos/OWNER/REPO \
  -d '{"has_issues": true, "allow_forking": false}'
第四步：用GitHub CLI批量检查所有私有仓库：
gh repo list --private --json name,owner --limit 1000 | jq '.[] | "\(.owner.login)/\(.name)"' | xargs -I {} gh repo view {}

更根本的防御是在本地备份你的代码，挣脱单一平台的束缚。GitLab和Gitea都是值得考虑的替代方案。

■ 微软账号门
https://www.windowscentral.com/microsoft... · https://news.ycombinator.com/item?id=475... · https://www.reddit.com/r/windows/comment...

2026年3月，Windows Central爆出一则令开发者社区震动的消息——微软内部正爆发一场关于Windows 11是否应继续强制用户在安装过程中绑定Microsoft Account的激烈争论。这场"内战"在外网炸出420 points、347条评论，是本期话题中声量最高的事件之一。反对者认为强制账号绑定侵犯用户隐私，支持者则声称这是"改善用户体验和安全性"的必要举措。两种叙事背后，是微软从软件公司转型为订阅服务平台的深层焦虑。

Windows 11从发布伊始就在账号绑定问题上展现了极度强硬的姿态。与Windows 10允许本地账户绕过的做法不同，Windows 11在OOBE（开箱体验）流程中强制要求登录Microsoft Account，否则无法完成安装。这一设计引发了大量用户和隐私倡导者的强烈抗议，甚至催生了大量"离线账户绕过教程"的病毒式传播。 微软的逻辑链表面自洽：账号绑定 → 云端同步 → 跨设备体验一致 → AI功能个性化。Copilot+PC的战略更需要用户账号体系作为底层支撑。但这套逻辑忽略了一个根本问题：用户是否有权选择不使用云服务而仅运行本地操作系统？ 更深层的矛盾在于微软的商业模式转型压力。Windows曾经是微软的现金流业务，如今已被云和订阅服务战略边缘化。强制账号绑定本质上是将操作系统从一次性销售品转变为持续性服务入口——用户不登录账号，Microsoft 365、AI功能、OneDrive等高利润服务的变现链路就无法启动。 这场内战的结果将决定Windows未来十年的用户关系。如果微软继续推行强制账号，可能引发类似当年Windows 10免费升级潮后的用户信任危机，大量注重隐私的用户可能转向Linux或等待苹果ARM生态的进一步成熟。反对派的声音能否转化为产品路线的调整，值得密切关注。

■ A Faster Alternative to Jq——JSON处理工具的性能革命

一个名为jsongrep的工具在HN炸出356 points、225评论，它自称是jq的替代方案，在大数据量JSON处理场景下有数量级的性能提升。jq是Unix/Linux下处理JSON数据的标配工具，但其解释型语言的设计在处理超大JSON文件时性能瓶颈明显。jsongrep使用编译型语言重写了核心解析器，并针对现代CPU缓存和SIMD指令做了优化。在实测中，处理100MB级别的JSON文件时，jsongrep比jq快20-50倍。对于每天和数据管道打交道的工程师而言，这个工具的实用价值极高——它是真正的"效率武器"，不是炫技项目。
