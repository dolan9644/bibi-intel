🎯 核心阵地
核心阵地一｜可验证 Agent 堆栈上桌：代码生成退潮，证据机器才是新印钞机

背景溯源：过去两年，coding agent 的大部分表演都停在“我能替你写代码”。听起来很爽，像把一个不用发股票的初级工程师塞进命令行。但真实软件团队的痛点从来不是“能不能吐一段 diff”，而是这段 diff 在谁的机器上跑过、怎么复现、失败路径有没有视频、权限有没有越界、reviewer 为什么应该相信它。Peter Steinberger 展示的 ephemeral crabboxes + WebVNC 流程，把这层皮直接掀开：agent 能在 Linux、Windows、macOS 的短命环境里搭建精确状态，复现 issue，测试修复，并把视频贴回 PR。原文链接：https://x.com/steipete/status/2051557150040711425。Guillermo Rauch 同时发布 open-source agent orchestrator deepsec，用 npx deepsec 对 OSS 项目做深度安全 review，声称已经在内部和主要开源项目里抓到 critical issues。原文链接：https://x.com/rauchg/status/2051386798899888539。再叠上 Reflex 那篇“Computer Use 比结构化 API 贵 45 倍”的冷水，原文链接：https://reflex.dev/blog/computer-use-is-45x-more-expensive-than-structured-apis/，今天的主线就清楚了：会点屏幕不是智能，能低成本、可审计、可复现地完成任务，才是生产力。

完整分析：这不是又一轮“AI 写代码更快”的老掉牙叙事，而是 agent 从生成器升级成证据系统。传统 CI 只能回答脚本过没过，不能回答用户路径是否真的没炸；传统静态扫描能报一堆洞，但经常不负责复现和上下文判断；传统 RPA 能点 UI，但一碰到弹窗、状态污染、跨平台差异就开始原地抽风。新的可验证 Agent 堆栈至少有三层：第一层是短命沙箱，环境必须可丢弃、可重建、可录屏，最好还能把系统状态快照和网络权限写进证据包；第二层是工具编排，agent 不能把视觉点击当主路径，而要优先走 CLI、API、测试框架、包管理器、审计器和日志接口；第三层是证据回传，PR 里不能只贴 diff，还要贴复现命令、失败截图、修复后视频、权限清单和测试增量。只要这条链路打通，reviewer 的工作就从“猜这个模型是不是胡来”变成“检查证据是否足够”。这才是真正能卖钱的部分：不是替工程师敲键盘，而是替团队降低不确定性。

代码/配置演示：生产环境最小可用骨架应该长这样，别再把 agent 当许愿池。

- name: Upload evidence

大佬链接：Swyx 的 Latent Space 访谈 OpenAI 研究员 Alex Lupsasca，主题是 Doing Vibe Physics 与 GPT-5.x 形成过程，原文链接：https://www.latent.space/p/lupsasca。它最值得咀嚼的不是“GPT-5.x”这几个字，而是前沿模型研发越来越像实验物理：用行为诊断、训练信号、失败样本和调参直觉去定位模型的脾气。Peter 的可视复现、Rauch 的 deepsec 和 Lupsasca 的 vibe physics 其实是一条线：模型不是神谕机，模型是实验对象；agent 也不是魔法师，agent 要被关进实验室，留下轨迹，交出证据。

实战优劣势分析：优势很硬。第一，PR 质量从“模型说它修好了”升级到“这里是复现、修复和回放”；第二，安全审查从一次静态扫描升级成多轮探索、复现和报告；第三，短命沙箱降低环境污染和密钥泄露风险；第四，视频证据能让 reviewer 快速理解 UI 行为，不用靠脑补补洞。劣势也别装看不见。沙箱、录屏、跨平台镜像都会烧钱；agent 安全 review 如果没有范围限制，会把 reviewer 淹死在噪音里；浏览器和 Computer Use 适合补缺口，不适合当高频主路径；一旦证据包里混入敏感日志，公司就会从“自动化审计”直接滑进“自动化泄密”。

Dolan's 锐评：代码生成已经不性感了，真正值钱的是“能自证没炸”。过去 AI 公司卖的是“我能写”，现在必须卖“我写完能验尸”。钱往哪流？流向沙箱、回放、权限、审计、连接器和证据层。还在把 Computer Use 当万能钥匙的团队，本质是在拿云推理成本给自己的毛利挖坟。GUI 自动化是创可贴，不是骨科手术；结构化接口才是长期现金流。Agent 时代的赢家不是嘴最甜的聊天框，而是能把模型锁进生产纪律里、让每一次失败都留下尸检报告的平台。

核心阵地二｜AI Character 与行业 Agent 撞车：拟人化留存和合规现金流只能活一个

背景溯源：Latent Space AINews 用 Clippy vs Anton 讨论 AI character 的“他者”与“工具”定位冲突，原文链接：https://www.latent.space/p/ainews-the-other-vs-the-utility。表面看这是产品人格之争：AI 要不要像一个有脾气、有记忆、有存在感的对象？实际看，这是权限、记忆、主动性、披露和责任归属的系统战争。同一条情报提到 Sierra 约 10 亿美元融资、估值 150 亿美元，客服 agent 仍然是资本最爱抽血的场景。另一边，Anthropic 发布金融服务与保险 agents，官方链接：https://www.anthropic.com/news/finance-agents，把战场放到投研、审计、估值、承保、理赔、KYC 和合规流程。GLM-5V-Turbo 论文则把叙事推到底层：多模态 agent 不能只是语言模型外接摄像头，而要把视觉、语言和工具使用做成原生能力。论文链接：https://arxiv.org/abs/2604.26752。

完整分析：今天 AI 产品正在分叉。一条路是 character，追求留存、亲密感、品牌人格和用户习惯；另一条路是 regulated workflow，追求合规、审计、权限和结果可追责。前者像消费互联网的老药方：多记一点，多主动一点，多像人一点，用户就多回来一点。后者像金融圈的冷账本：你为什么调用这个数据源，凭什么给这个结论，谁批准这个操作，错了谁背锅。很多创业公司最贪心，想两边都吃：对 C 端讲“像伙伴”，对 B 端讲“可审计”，对投资人讲“平台入口”。这像把会撒娇的客服包装成投行副总裁，融资路演很好听，生产环境很难闻。

AI character 真正危险的地方，不是语气像不像人，而是系统边界会被人格稀释。只要它有长期记忆、主动提醒、情绪反馈和任务权限，用户就会把它从工具看成“他者”。这能提高留存，也会制造依赖、操纵和责任错觉。行业 agent 正相反，它必须把人格压低，把动作抬高，把留痕做厚。金融与保险不是让 Claude 显得更聪明的秀场，而是要求 agent 像合规员工一样工作：输入来源要明，推理依据要留，权限边界要窄，输出风险要标。GLM-5V-Turbo 的多模态路线提醒了另一件事：下一代 agent 要在视觉环境里操作真实流程，屏幕理解、表格识别、工具调用、长程任务都会进入同一个执行图。那时“人格”只是 UI 糖衣，真正决定价值的是任务链路能不能扛住审计。

代码/配置演示：行业 agent 的入口不是一句“你是聪明助手”，而是策略文件、权限面和审计日志。

大佬链接：Swyx / Latent Space 的 AI character 讨论原文链接：https://www.latent.space/p/ainews-the-other-vs-the-utility。Swyx 这类 practitioner 社区真正抓住的问题是，AI 产品的人格不是文案层，而是权限层：记忆、主动性、称呼、拒绝方式、退出机制、用户能不能删除历史，这些才是 character 的骨架。Anthropic 的金融 agents 官方文章则代表另一条资本路径，链接：https://www.anthropic.com/news/finance-agents：高客单价、强合规、深集成，用“能进流程”替代“像个人”。

实战优劣势分析：character 路线的优势是获客和留存直接，用户一旦形成互动习惯，迁移成本会被情感和记忆悄悄抬高；适合陪伴、教育、轻客服和品牌型助手。劣势是监管雷区密集，一旦涉及未成年人、健康、金融、情感依赖或默认主动触达，就会把公司拖进披露、操纵和责任泥潭。行业 agent 的优势是钱厚、需求硬、ROI 能算，尤其是金融、保险、法务、合规和企业运维；劣势是销售周期长、数据接入脏、审计要求高，模型回答得再漂亮，也敌不过一个权限口没打通。多模态 agent 的优势是能进入真实工作界面，劣势是风险面暴涨：它看到了什么、点了什么、保存了什么，都得能解释。

Dolan's 锐评：AI character 是留存毒药，行业 agent 是合规苦药。一个让用户觉得“它懂我”，一个让审计觉得“它没乱来”。创业公司最容易死在中间：既想靠人格绑住用户，又想靠企业合同装严肃，最后变成一个会讲情绪价值的流程事故源。资本今天追 Sierra、追金融 agents，本质不是爱 AI，而是爱高频客服和高客单价白领流程。别被拟人化迷魂汤灌晕：在真正付钱的市场里，AI 的脸不重要，账本、权限和责任链才重要。

板块总结

Dolan's 锐评：巨头们今天不再只比谁的模型更大，而是在比谁能控制默认入口、降低推理成本、吃下高客单价流程，并把 agent 关进审计笼子。Google 往浏览器塞本地模型，Anthropic 钻金融保险，OpenAI 谈实验物理，Vercel 推安全 orchestrator，方向都很赤裸：能力叙事退场，单位经济和责任链上桌。行业不会为“像人”长期买单，只会为“便宜、可查、能赔少点钱”买单。谁还在喊玄学智能，谁就是下一轮融资市场的背景音。
