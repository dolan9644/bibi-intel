Anthropic 是目前唯一一家公开系统提示词的前沿实验室，这件事本身就够讽刺的——其他家把系统提示词当核弹发射密码捂着，Anthropic 倒是大大方方把源码贴出来。Simon Willison 这位 T0 级独立 AI 实践者，第一时间做了 4.6 和 4.7 的 delta 分析，结果一出来整个 AI 工程师圈都在偷偷对账。

系统提示词不是简单的"你是一个有帮助的 AI"，它直接决定了 Claude 的拒绝阈值、指令遵循模式、agentic 循环中的默认行为。Minor version 的小改动往往比大版本升级更能坑死人——因为没人会为 4.6→4.7 这种版本号尖叫，但你的 agent 行为可能已经悄悄变了个底朝天。 Simon 的分析揭示了几个关键变化： - 拒绝边界发生了微调，某些以前会拒绝的场景现在放行了 - Agentic 任务中的默认追问深度被削弱了，可能是为了减少 token 消耗 - 代码生成质量在边界情况下的偏好发生了漂移 这意味着什么？意味着所有在生产环境跑 Claude 4.6 agent 的团队，升级到 4.7 之后行为可能不一致，但你根本不知道为什么。传统软件有 changelog，AI 模型的行为 changelog 在哪儿？
