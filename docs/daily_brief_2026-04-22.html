Agent 工作台开始吃掉聊天订阅，开源底座与多模态入口同时改写估值锚
🎯 核心阵地
一、聊天框退位，工作台上桌：Claude Cowork 和 ChatGPT Images 2.0 正在把入口战争从模型参数表，狠狠干成工作流税表

过去一年，AI 圈最廉价的叙事，就是拿一堆 benchmark 截图当圣经，再配上几句“能力全面提升”就开始四处画大饼。现在这套戏法开始不够用了。真正值钱的，不再是某次榜单上多刷了 1.7 分，而是谁先把模型塞进用户每天都回来的工作台。Anthropic 这次把 Cowork 下沉到所有付费档，再加上 Live Artifacts 标签页和版本历史，本质上不是功能加法，而是把 Claude 从一次性聊天消耗品，改造成有记忆、有留存、有回访理由的项目容器。OpenAI 这边的 ChatGPT Images 2.0 则是另一条线，它把图像生成从独立玩具房拖回主聊天流，把“出图”重新定义成“对话中的一段可迭代操作”。这两个动作看着像不同赛道，实际上是同一件事：入口从问答，升级成持续生产。

Simon Willison 的测试最有参考价值，不是因为他替 OpenAI 吹喇叭，而是他没玩花活，直接拿高约束提示词去拷打可控性，验证“浣熊和业余电台”这种具体场景能不能听懂人话。原文链接：https://simonwillison.net/2026/Apr/21/gpt-image-2/#atom-entries
Anthropic 官方关于 Cowork 与 Live Artifacts 的发布链接：https://x.com/claudeai/status/2046328622869344429

这里面的真钱逻辑很简单。第一，持久化意味着留存。用户过去在聊天框里今天问完明天就散伙，现在项目、文稿、表格、原型、脚本都能留在原地，版本还能回滚，SaaS 味一下就上来了。第二，原生多模态意味着调用频次变高。以前图像生成像“逢年过节开一次”的烟花功能，现在它进入连续编辑循环，等于把生成成本和用户黏性一起绑定。第三，工作台化会重排估值锚。资本以前爱按 token、月活、推理成本讲故事，接下来更愿意看项目驻留时长、回访率、团队席位扩张率，因为那才像真钱生意。

更狠的是，这一波还会改变团队协作方式。过去很多人把模型当“高级搜索框”，问一嘴，复制一点，回 IDE 继续自己撸袖子单干。现在如果 Artifact 可以跨会话存活，图像、文本、原型都能在同一个界面里来回修，那么 AI 工具就不只是副驾驶，而是在偷偷吃掉文档系统、原型工具、轻量知识库，甚至部分低配 IDE 的饭碗。谁先把这些碎片收编进一个低摩擦工作台，谁就先把用户锁进自己账本。所谓“模型能力领先”，如果不能沉淀成工作流资产，最后也只是高成本烟火秀。

这对开发团队意味着什么？意味着你不能再用“试用一下模型效果不错”这种幼儿园级评估标准。你得盯四个指标：一是会话恢复能力，二是产物可追踪性，三是多模态编辑链路的失真率，四是权限和审计。没有这四样，所谓 AI 工作台只是把草台班子搬进浏览器。尤其是图像链路，一旦进入聊天原生流程，敏感素材、设计稿、客户图像就会被大面积喂进系统，没有审计和脱敏，后面出一次事故，法务就会教你什么叫真钱回撤。

实战优势很清楚。第一，团队协作成本降低，尤其是产品、设计、运营之间来回扔截图和文档的低效动作会被砍掉一截。第二，版本历史把“刚才那版更好”这种废话变成可回滚事实。第三，多模态原生编辑会显著提升提示词的上下文连续性，少一点“重新开题”的脑裂。第四，工作台一旦形成组织习惯，工具切换成本会大幅抬升，供应商就有了续费筹码。

但坑也很实在。第一，持久化带来数据责任，存得越多，雷越大。第二，多模态工作台看起来炫，实际上对权限、存储、审计、成本控制提出更高要求，小团队很容易被表面效率骗进去，最后被账单和合规反杀。第三，工作台型产品会天然追求平台化，平台化一旦过头，性能、透明度和可迁移性就会开始掉链子。第四，图像能力并不自动等于生产能力，很多“好看”的 demo 在真实需求里一碰到品牌规范、版式约束、法务水印，立刻原形毕露。

Dolan's 锐评：这波不是谁的模型更聪明，而是谁先把聊天框包装成收费地皮。Cowork 和原生图像编辑都在干同一件脏活，把一次性问答改造成可计费、可锁仓、可复购的工作流资产。谁还盯着排行榜高潮，谁就是拿实验室成绩单给 SaaS ARR 做法事。

二、开源底座开始掀桌：Kimi K2.6、Claude Code 提价和平台安全税，其实是同一张利润表上的三列数字

如果说前一条主线讲的是入口，那么今天更值得盯的是底座定价权。Latent Space 把 Moonshot Kimi K2.6 直接抬到“对打 Claude Opus 4.6，顺手盯防 DeepSeek 下一轮”的位置，信号非常硬。原文链接：https://www.latent.space/p/ainews-moonshot-kimi-k26-the-worlds
这不是媒体嘴炮，而是全球开发者心智已经默认中国开源模型不再只是性价比替补，而是有资格轮流冲击前排席位。与此同时，Anthropic 把 Claude Code 从 Pro 档拿掉，定价页面摆明了告诉市场：代码代理不是慈善，重度编程推理要么单独付费，要么滚去更高客单价。相关链接：https://claude.com/pricing
再补上一刀，Vercel OAuth 与环境变量暴露风险被拉出来公开鞭尸，说明模型能力越强，连接真实系统越深，安全税就越不可能装死。相关讨论入口：https://www.trendmicro.com/en_us/research/26/d/vercel-breach-oauth-supply-chain.html

把这三件事放一起看，逻辑一下就透了。第一，开源模型把底价打下来，逼闭源厂商重新切套餐、拆能力、搞分层收费，不然毛利率会被重型用户狠狠干穿。第二，代码代理这种高调用、高上下文、高容错要求的场景，本来就是推理成本黑洞，任何一家厂商只要还想维持漂亮财报，就一定会把它从“通用订阅福利”中剥离出去。第三，安全问题把“低价可用”再砍一刀，因为真正落到企业生产里，除了模型费，还有权限隔离、密钥管理、审计、回滚、合规这些看不见的硬成本。

Swyx 这次给 Kimi 的定位，最值得咀嚼的不是“世界最强”这种媒体辣条标题，而是它把竞赛规则改成了迭代速度和开发者心智争夺。今天开源模型真正厉害的地方，不是一次评测爆种，而是发布频率快、生态接入快、开发者试错成本低。只要一款模型能在真实项目里达到八成以上可用，市场就会立刻拿它去给闭源高价模型压估值。换句话说，闭源厂商以后卖的不只是智商税，而是稳定性税、品牌税、法律责任税、以及企业采购安心税。

这对团队采购策略的启示非常直接。别再问“最强模型是谁”，那是论坛键盘侠的问题。真正该问的是：哪些任务能用开源或低价模型吃掉，哪些任务必须留给高安全、高可靠的闭源端点，哪些任务干脆应该在隔离环境本地跑。尤其是代码代理，别被演示视频催眠。只要代理开始碰仓库写权限、CI 密钥、生产环境变量，它就不再是聊天机器人，而是半自动运维刀具。刀锋够快是优点，砍到自己手也是现实。

生产路由建议直接按任务和风险分层，而不是按厂商品牌崇拜分层：

SECONDARY_MODEL=claude_opus
LOCAL_MODEL=qwen_or_other_local
TASK_CLASSIFIER=enabled
SECRETS_PROVIDER=vault
OIDC_REQUIRED=true
EPHEMERAL_RUNTIME=true
STATIC_ANALYSIS_REQUIRED=true
HUMAN_APPROVAL_ON_WRITE=true

AUDIT_LOG_IMMUTABLE=true

实战优势非常明显。第一，开源和低价模型可以把大批中等价值任务的边际成本狠狠干下来，让团队别再用法拉利去买菜。第二，闭源高端模型只保留给真正需要稳定性和复杂推理的场景，成本结构更健康。第三，安全与审批前置后，代码代理至少不会一脚把你送进事故复盘会。第四，多模型路由还能反向压供应商，让你不至于被一家平台当韭菜反复收割。

但缺点也不能装看不见。第一，多模型治理复杂度高，监控、评测、回退机制没搭好，最后可能比单一厂商更乱。第二，开源模型的速度和性价比虽猛，但在长链路可靠性、合规责任、企业支持上仍可能掉链子。第三，安全隔离一旦做真，短期内会拖慢交付节奏，很多业务老板会嫌麻烦，可一旦不做，迟早要用事故补课。第四，提价和套餐切割会持续发生，任何把供应商宣传页当合同附件的团队，后面都会被财务部追着问责。

Dolan's 锐评：Kimi K2.6 掀的是底价，Claude Code 提的是客单价，Vercel 漏的是隐藏成本。三者拼起来就一句话，AI 基建从来不是比谁最会讲故事，而是比谁在利润、性能、安全这三根钢丝上摔得更晚。还把“便宜又全能”当真理的团队，基本等于拿预算给别人交学费。

板块总结

Dolan's 锐评：今天大厂和平台都在做同一件事，先把入口抢下来，再把高成本能力拆包定价，最后把安全和治理成本悄悄甩给客户。模型性能还会涨，但真正决定谁赚到钱的，是谁能把留存、套餐和风控捏成一个完整漏斗。只会吹模型神话，不会算单位经济和运维风险，迟早被市场狠狠干醒。
