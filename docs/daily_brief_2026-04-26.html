代理型劳动开始吞噬软件工位，价格战正把模型底座的估值打回原形
🎯 核心阵地
一、Codex 不再卖模型，它在卖开发者驾驶舱

过去两年，AI 编程圈最大的一场幻觉，是大家都以为谁把 benchmark 做得更漂亮，谁就能坐上下一代开发平台的王座。现在这层窗户纸被 OpenAI 自己捅破了。GPT-5.5 API 发布并不是一条单纯的模型升级公告，它和 Codex 应用重构被绑在一起推进，实质是把模型能力、任务执行、项目上下文、交互界面和交付责任塞进同一个壳层里。换句话说，OpenAI 不满足于继续卖 token，它要把开发者一天里最值钱的工位变成自己的收费闸机。过去卖的是发动机，现在卖的是整台驾驶舱，顺手把方向盘也焊死。

这件事为什么危险，或者说为什么值钱，得从 agent coding 的真实演化说起。第一阶段是补全时代，AI 只会在你键盘边上当一个高配自动联想。第二阶段是问答时代，模型能解释代码、补 bug、写测试，但始终像个嘴很甜的外包实习生。第三阶段才是今天正在爆发的代理执行时代，用户不再关心模型会不会说，而是盯着它能不能独立完成需求、跑通依赖、提交文件、修复报错、收尾交付。Swyx 在 Latent Space 的标题直接点破了这个趋势，GPT 5.5 and OpenAI Codex Superapp，这不是写给技术宅看的标题，这是写给投资人和平台操盘手看的。只要 superapp 入口拿下，外围那些只包了一层聊天壳、插件壳、IDE 壳的产品，估值就会开始冒烟。

更扎心的是，一线 builder 的反馈已经从情绪赞美切换成了工作流分工。Peter Yang 拿 F-Zero 测试这类完整游戏生成任务做验证，结论不是 GPT-5.5 更会聊天，而是 GPT-5.5 加 Codex 这套组合终于能把能跑的东西交出来。Dan Shipper 的判断更赤裸，他强调的不是计划质量，而是模型从会写计划转向直接把事做完。Aaron Levie 又补上了最反常识的一刀，AI 并没有让人少干活，反而因为探索成本坍塌，人会尝试更多路径，本来十分钟的小修补，膨胀成三小时的产品试验。Nikunj Kothari 则把行业真相说成一句大白话，Claude Opus 负责 planning 和前端构思，OpenAI Codex 负责执行和落地。注意，这套分工一旦成立，护城河就不再是单模型参数规模，而是谁能把不同模型安插进最顺手的生产位置。

于是，真正的战争线已经很清楚。OpenAI 想做的是开发者操作系统，不只是模型供应商。它把高价模型和高频工作流入口打包，目的是提高每开发者收入，把价值从一次调用抽水改成长期平台税。对独立 AI IDE 来说，这是一场标准的渠道绞杀。你本来还能靠更轻的界面、更灵活的插件、更便宜的包月活着，一旦底层模型厂商自己下场做驾驶舱，你就会发现自己其实只是给别人养用户习惯。对企业来说，诱惑也很大，因为一体化平台意味着更少的 glue code，更短的接入链路，更快的 PoC 速度。但代价也摆在桌上，权限边界、代码审计、回滚记录、软件供应链责任都会被一起抬上来。模型一旦从建议者升级为执行者，任何一次误删、误部署、误依赖注入，都会从产品事故升级为组织事故。

再给一段更像实战的执行链路：

import requests

TASK = {
"goal": "为支付服务增加幂等键校验，并补齐回归测试",
"repo": "git@company.example/payments.git",
"constraints": ["不得改动 public API", "必须通过 pytest 和 contract test"]
}

plan = requests.post(
"https://router.internal/plan",
json={"model": "claude-opus", "task": TASK}
).json()

execution = requests.post(
"https://router.internal/execute",
json={"model": "gpt-5.5", "tool_mode": "codex", "plan": plan, "sandbox": "workspace_only"}
).json()

review = requests.post(
"https://router.internal/review",
json={"model": "gpt-5.5", "patch": execution["patch"], "checks": ["tests", "security", "rollback"]}
).json()

print(review["status"])

这套打法的优势很硬。第一，规划和执行拆开后，模型各干各擅长的活，吞吐量明显上去。第二，审计和回滚被制度化，不再靠人类事后擦屁股。第三，平台入口一旦稳定，团队协作会从提示词小作坊升级成任务流水线。缺点也同样扎眼。第一，供应商绑定会越来越深，迁移成本和席位成本一起上升。第二，日志、权限、知识产权归属会更复杂，法务和安全团队迟早要下场。第三，一体化 superapp 很可能把开发者训练成依赖平台肌肉记忆的租客，爽是很爽，议价权却被抽干。

Dolan's 锐评：OpenAI 这波不是发新版模型，是来收开发者过路费的。谁还把 Codex 看成一个代码功能点，谁就跟还把 iPhone 看成功能机一样，认知停在上个朝代。以后最贵的不是最强模型，而是谁握住任务入口、执行闭环和组织日志。模型公司一旦学会吃工作流，外围壳层创业公司就只剩两条路，要么卷垂直场景，要么等着被平台当配菜。

二、DeepSeek V4 把高端模型做成期货贴水，LiteParse 把文档入口抢回本地

如果说 OpenAI 在上游工位抢入口，那 DeepSeek 干的是另一件更坏的事，它在拆穿前沿模型的定价神话。Simon Willison 当天直接把判断写进标题，DeepSeek V4 almost on the frontier, a fraction of the price。别小看这句判断，这不是国产自嗨式捷报，而是英语世界最敏锐的一批独立实践者开始承认，中国模型已经不只是追赶者，而是闭源头部定价权的拆弹手。根据 Simon 的整理，DeepSeek-V4-Pro 和 DeepSeek-V4-Flash 都给出 1M token context，Pro 总参数 1.6T，激活 49B，Flash 总参数 284B，激活 13B。更狠的是价格，Flash 输入 0.14 美元每百万 token，输出 0.28；Pro 输入 1.74，输出 3.48。这不是便宜一点，这是直接把高端区间砍出一道口子，让之前靠品牌溢价硬撑的前沿底座瞬间显得像奢侈品税。

但价格战真正有杀伤力的前提，不是便宜，而是够用并且可落地。Simon 点出的重点不是 DeepSeek 绝对登顶，而是已经足够接近 frontier，进入主流替代名单。这里面最关键的信号有两个。第一，DeepSeek 把长上下文效率继续做薄，1M token 场景下，Pro 单 token FLOPs 只有 V3.2 的 27%，KV cache 只有 10%；Flash 更狠，分别只有 10% 和 7%。第二，它用 MIT 许可和开放权重，把模型采购从品牌信仰往性价比纪律推。你可以不喜欢它的地缘背景，但 CFO 会喜欢这张账单，工程团队会喜欢更大的上下文，买方市场会喜欢终于有人出来踹翻闭源模型的高毛利酒桌。

这条线为什么要和 LiteParse 放在一起看，因为便宜模型如果没有便宜上下文入口，最后还是会死在摄取链路上。LiteParse for the web 的意义，不在于它是个酷炫 demo，而在于它把文档 AI 里最容易被忽视的一段脏活累活，也就是 PDF 提取、OCR、版面排序、bounding box 标注，直接搬回浏览器本地。Simon Willison 的文章把这件事写得很明白，LiteParse 不靠模型胡诌，而是靠 PDF.js、OCR 和空间解析把文本按更可信的顺序剥出来。更关键的是，浏览器版意味着敏感文档不必先上传到云端排队挨刀，再返回给你的 RAG 管线。对于政企、金融、法务、研究团队来说，这不是小修小补，这是上下文入口的主权回归。

把这两条线拼起来看，行业正在出现一套更现实的新栈。上游用 DeepSeek V4 之类的近前沿低价模型消化大上下文，中游用 LiteParse 这种本地优先工具把脏文档洗干净，下游再由 agent 框架做路由、问答、执行和引用验证。钱往哪流也就很明白了，单纯卖聊天窗口的会越来越难，卖低成本高质量上下文、卖可审计工作流、卖本地优先隐私能力的，反而会吃到预算。Google 想拿 400 亿美元继续绑 Anthropic，本质也是同一逻辑，模型和算力要深绑；而 DeepSeek 和 LiteParse 这种组合，干的是另一种反向绞杀，拿低价开放和本地入口，把平台税往下打。

生产配置演示，给做文档问答和企业检索的团队一个能落地的骨架：

pipeline: local-first-rag
inputs:
pdf_dir: ./incoming
max_file_size_mb: 200
extract:
engine: liteparse
mode: browser_or_cli
ocr: auto
output: json
keep_bounding_boxes: true
chunking:
strategy: section_plus_layout
target_tokens: 1200
overlap_tokens: 120
routing:
- if: context_tokens > 200000
model: deepseek-v4-pro
- if: latency_priority == high
model: deepseek-v4-flash
- if: answer_requires_final_review
model: human_signoff
citations:
visual_citation: enabled
answer_must_include_page: true
security:
upload_raw_pdf_to_cloud: false
pii_redaction: required

对应的执行脚本可以这么写：

npm i -g @llamaindex/liteparse
lit parse ./incoming/policy.pdf --json > ./work/policy.json

import json
import requests

with open("./work/policy.json", "r", encoding="utf-8") as f:
doc = json.load(f)

resp = requests.post(
"https://router.internal/rag-answer",
json={
"model": "deepseek-v4-pro",
"question": "列出该政策中所有涉及数据出境审批的条件",
"document": doc,
"must_include_visual_citations": True
}
)

print(resp.json()["answer"])

这套打法的优势非常现实。第一，单位成本骤降，大上下文场景终于不是只有巨头预算能玩。第二，文档在本地提取，合规和隐私阻力会小很多。第三，bounding box 和可视引用能压住一部分 RAG 幻觉，至少知道模型胡说时是哪一页在背锅。劣势也别装看不见。第一，LiteParse 这类本地解析对复杂表格、手写件、扫描件仍有上限，真碰上脏到离谱的企业文档，还是得上更重的云解析。第二，DeepSeek 的企业级落地在欧美会继续被地缘信任和数据边界牵制。第三，开源和低价会让试错门槛更低，但也会让市场更快卷成红海，最后拼的还是分发和系统集成，而不是单点模型感人故事。

板块总结

Dolan's 锐评：这一轮绞肉机的底层逻辑非常脏但很真实，巨头不再满足于卖模型，也不甘心只卖云，他们要把入口、算力、日志、结算权一起吃下。与此同时，DeepSeek 这种低价近前沿玩家开始把闭源高毛利打回现实，LiteParse 这类工具又把上下文入口抢回本地。上游在收编，中游在路由，下游在本地化，谁还沉迷单点模型神话，谁就会在下一轮预算审判里被当成费用中心清理掉。
