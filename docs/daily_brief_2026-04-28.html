一、会用电脑才配叫智能体，Coding Agent 正从玩具脚本升格为新一代数字牛马总包

背景溯源：
过去两年，所谓 AI 编程大多还停留在花哨补全，像是给 IDE 喂了一层糖衣炮弹，演示时看着顺滑，真上产线就开始原形毕露。它会续写函数，却不会处理依赖冲突；会生成 Dockerfile，却不敢自己把镜像跑起来；会写单元测试，却看不懂 CI 爆红到底是环境变量、权限、网络还是缓存炸了。Guillermo Rauch 这次把话挑明，coding agents 真正的护城河不是会写几段代码，而是能否像熟练工程师一样，接管 bash、文件系统、包管理、服务安装、故障回滚这些脏活累活。原帖链接：https://x.com/rauchg/status/2048523195305902341

这句话值钱的地方，不在于鸡血式喊超级智能，而在于它戳穿了行业里一堆假繁荣。过去很多产品卖的是“像工程师”，本质上只是“像个很会 autocomplete 的实习生”。真正能掏企业预算的，不是会在聊天框里写出漂亮函数，而是能把一个工单完整闭掉。也就是说，竞争不再是模型 benchmark 上几分的差距，而是端到端 computer use 能力，谁能在终端里活下来，谁才有资格谈下一轮增长。

这里面最关键的变化，是评估口径从“会不会答”转成“能不能干完”。智能体如果要代替人类完成软件交付，必须跨过四道坎。第一道坎是环境理解，它得知道项目依赖什么、目录结构怎么组织、服务启动路径在哪。第二道坎是工具编排，不只是调用一个编辑器，而是能在 shell、包管理器、浏览器、日志系统之间来回切。第三道坎是容错恢复，命令跑挂了不能装死，得会重试、降级、换源、查日志。第四道坎是安全边界，企业不会让一个会乱删库的高智商脚本直接进生产。

GitHub Copilot 改成 usage-based billing，等于又补了一刀，直接把这个行业从卖座位拉回卖算力、卖工作量、卖任务完成度。来源：https://github.blog/news-insights/company-news/github-copilot-is-moving-to-usage-based-billing/ 这意味着厂商自己也知道，AI 编程不是 SaaS 老路子，不能再靠按人头包月糊弄。轻度用户和重度用户消耗完全不是一个量级，长任务、多轮修复、工具调用、上下文回放，这些都是真金白银烧出来的。谁再跟你说智能体只是一个更聪明的编辑器插件，那就是拿旧世界的席位逻辑去解释新世界的算力账单，纯属会计失真。

再往下看，macOS 27 网络栈变化和 NPM 官网宕机这种看似零碎的事件，恰恰是智能体落地最容易踩雷的地方。来源分别是：https://eclecticlight.co/2026/04/23/networking-changes-coming-in-macos-27/ 与 https://status.npmjs.org 。一旦操作系统网络权限模型收紧，或者公共包仓库临时抽风，所谓“会写代码”的 agent 立刻变回纸老虎。人类工程师还能靠经验手动切换镜像、改代理、查系统权限、补 hosts、离线装包，很多 agent 现在还做不到。说白了，电脑使用权不是一个附加功能，而是智能体从 demo 走向劳动生产力的生死线。

大佬的新感悟与链接：
Guillermo Rauch 的核心判断是，未来 coding agents 的胜负手不在单点生成，而在完整计算机环境掌控力。链接：https://x.com/rauchg/status/2048523195305902341
Simon Willison 另一边给了一个治理视角，他追踪 OpenAI 与微软 AGI 条款的死亡，间接证明行业已经从“神性叙事”切回“现实执行”。链接：https://simonwillison.net/2026/Apr/27/now-deceased-agi-clause/#atom-entries

实战优劣势分析：
优势一，能真正吞掉高频、低创造性、流程明确的工程杂务，比如修依赖、补测试、跑迁移、提 PR，直接给团队省掉一堆被上下文切换撕碎的时间。
优势二，工作留痕更完整。人类工程师半夜救火往往只留下几句 Slack 脏话，agent 反而能完整保留命令、diff、日志和失败路径，适合做知识沉淀。
优势三，适合和平台工程绑定。谁掌握仓库权限、运行沙箱、日志入口和成本路由，谁就掌握企业级 agent 分发权。
劣势一，对真实世界的脏环境仍然脆弱，网络抖动、私有仓库认证、系统兼容性、代理配置这些问题，依旧能把一堆“榜单第一”的模型打回小学水平。
劣势二，成本模型还没稳定。按使用量计费之后，企业很快会发现，放任 agent 长上下文乱跑，就是把预算喂给 GPU 账单。
劣势三，责任边界模糊。代码能回滚，错误决策未必能回滚，特别是当 agent 开始改基础设施与安全配置时，审计链必须比今天更硬。

Dolan's 锐评：
别再把 coding agent 当高级自动补全了，那套话术已经过季。真正能吃下预算的，不是会写几百行样板代码的花瓶，而是能在终端、网络、依赖和失败日志这堆垃圾堆里把活干完的数字包工头。谁先把 agent 训练成会用电脑、会认错、会回滚、会算账，谁就能把“AI 提效”从路演 PPT 变成财务表里的真钱。剩下那些还在卷模型分数的玩家，本质是在拿显卡烧香，指望用户继续为幻觉买单。

二、AGI 条款死了，微软和 OpenAI 从政治婚姻走向按桌结账，资本市场终于把科幻皮套撕下来

背景溯源：
Simon Willison 对 OpenAI 与微软历史合同里 AGI 条款的回顾，是今天最值得细读的行业信号之一。链接：https://simonwillison.net/2026/Apr/27/now-deceased-agi-clause/#atom-entries 早年那套设计，本质上是把“如果某天董事会认定 AGI 到了怎么办”写进商业合同，让一个模糊、带宗教色彩的技术概念，直接触发权益切换。听上去很前沿，实际上非常离谱。因为 AGI 从来不只是能力定义问题，它还是治理接口、收益开关、权力按钮。谁有资格说“到了”，谁就有资格改写分成。

如今这条款被视作事实上死亡，意味非常直白，行业已经懒得陪这种抽象戏法继续演了。市场不再关心你离 AGI 还有几个 benchmark，也不再迷恋董事会拍脑袋宣布“人类文明转折点”的神棍时刻。大家真正关心的是，算力谁掏钱，收入怎么分，模型往哪卖，云资源谁兜底，谁能拥有更多分发渠道，谁在企业客户面前拿主导权。换句话说，AI 从“叙事金融”重新落回“合同金融”。这不是退步，恰恰是成熟的开始。

Bloomberg 关于微软停止与 OpenAI 排他和收入分成安排的报道，把这个趋势钉死了。链接：https://www.bloomberg.com/news/articles/2026-04-27/microsoft-to-stop-sharing-revenue-with-main-ai-partner-openai 这等于宣告双方关系从战略绑定转入市场化重估。以前的逻辑是，微软用云、渠道和资本换 OpenAI 的排他性和增长红利；现在的逻辑变成，双方都觉得继续捆死不划算。OpenAI 不想一直背着单一云渠道的镣铐，微软也不想继续当那个无底洞式补贴算力、却未必吃到全部利润的冤种金主。

这背后至少有三层结构变化。第一层，模型公司开始追求分发多元化。只要能力足够接近，任何单一云绑定都会逐渐变成估值折扣。第二层，云巨头也在去风险化。它们不想让未来增长完全压在一家模型公司身上，尤其当生态里 Anthropic、Google、Meta、自研模型都在加速时，押单票的风险越来越像二级市场上满仓单边梭哈。第三层，企业客户也在变聪明，他们要的是稳定供给、价格可谈、可替换架构，而不是跟着某家叙事明星一起坐过山车。

这就是为什么 Manus 被中国阻止出售给 Meta 也要放在同一张图里看。链接：https://www.cnbc.com/2026/04/27/meta-manus-china-blocks-acquisition-ai-startup.html 当 AI 资产、人才、数据与工作流越来越接近国家能力，跨境并购就不再只是普通财务退出，而是战略资源流向问题。AI 创业公司过去幻想的大厂收购退出通道，未来会越来越像带刺铁丝网。资本市场会因此重新定价中国 AI 项目的退出难度，创业者也会被迫更早考虑本地化造血，而不是指望被美国巨头高价打包带走。

再看 Mercor 4TB 语音样本泄露，链接：https://app.oravys.com/blog/mercor-breach-2026 这个事件把另一条隐蔽但更脏的链路暴露出来。AI 不只是模型和云，它还有庞大的外包劳动力系统，标注、语音采样、评测、人工审核，这些人和数据过去经常被当成后台耗材。一旦泄露，受伤的不只是隐私，还有跨境合规、劳务责任、企业声誉和未来训练数据供应链的合法性。很多公司嘴上说自己做的是前沿 AI，实际底层却是一条拿便宜劳动力和模糊授权堆起来的数据血汗工厂。现在出事了，账就要重新算。

大佬的新感悟与链接：
Simon Willison 的判断最狠，他实际上在说 AGI 条款的死亡，不是技术理想倒塌，而是商业现实接管了舞台。链接：https://simonwillison.net/2026/Apr/27/now-deceased-agi-clause/#atom-entries
如果把 Guillermo Rauch 的观点并进来，会发现另一条暗线，未来话语权不是谁先喊出 AGI，而是谁先把智能体变成真实可计费、可交付、可治理的劳动单元。链接：https://x.com/rauchg/status/2048523195305902341

实战优劣势分析：
优势一，去排他与去分成，能让模型公司争取更大分发自由度，也让企业客户获得更多议价空间，不再被某一对联盟锁死。
优势二，合同市场化之后，AI 采购终于能回归正常商业逻辑，算 ROI、算 SLA、算迁移成本，而不是被 AGI 神话牵着鼻子走。
优势三，多供应商与多云架构会倒逼更成熟的治理层、路由层和审计层出现，这是企业真正愿意长期埋预算的部分。
劣势一，切换供应商并不便宜，接口兼容、提示工程、评测体系、访问控制、数据驻留都要重做，迁移痛感非常真实。
劣势二，条款更市场化后，价格战和利润争夺会更残酷，模型厂商的财务报表可能比技术路线先出问题。
劣势三，监管与国家安全因素越来越深度嵌入交易，跨境并购、跨境部署、跨境训练都可能在最后一公里被卡脖子。
